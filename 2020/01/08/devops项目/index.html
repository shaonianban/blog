怎么让项目运行起来

安装依赖 npm install
运行项目 npm run dev

文件结构分布

框架 Layout

路径：views > layout > Layout.vue
引用的组件：
views > layout > components > Sidebar
views > layout > components > Navbar.vue

登录 Login

路径：views > login > index.vue

代码构建

代码构建：views > code-manage > structureManage.vue
代码构建详情：views > code-manage > structureDetail.vue
引用到的相关组件
src > components > CircleIcon 环境小圆点
PubDialog 发布弹框
BuildForm 构建弹框
repoInfo 分支信息

代码发布

代码发布：views > code-manage > pulishManage.vue
代码发布详情：views > code-manage > pulishDetail.vue
引用到的相关组件
src > components > CircleIcon 环境小圆点
src > components > LogRecord 日志组件
PubDialog 发布弹框
BuildForm 构建弹框
repoInfo 分支信息

其中有一个组件：repoBots是在BuildForm中被引用，功能为让用户选择对应的分支或tag

用户权限管理

用户管理：views > user > userManage.vue

环境配置

运行本地开发环境： npm run dev
修改本地开发环境的地址：

修改生产开发环境的地址

打包：npm run build:prod

前端构建打包

前端代码是通过代码发布平台构建的，所以需要关注build-init > npm.sh 文件 此文件的命令会在服务器中执行
